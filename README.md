# ELKON Yard Calculator Bot

Telegram-бот для расчёта заказов бетона:

- принимает `7.15` и `7,15`;
- переводит yd³ в м³ для ELKON;
- рассчитывает материалы;
- делит заказ на циклы максимум по 0,5 м³;
- показывает каждый цикл отдельно;
- предупреждает о слишком маленьком последнем цикле;
- сохраняет историю в PostgreSQL.

## Команды

- `/start`
- `/history`
- `/order 15`

## Локальный запуск

1. Создайте PostgreSQL.
2. Скопируйте `.env.example` в `.env`.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Запустите:

```bash
uvicorn main:app --host 0.0.0.0 --port 10000
```

Для локальной работы webhook должен быть доступен из интернета. Для обычного локального теста удобнее сразу развернуть проект на Render.

## Развёртывание через Render Blueprint

1. Загрузите файлы в GitHub.
2. В Render выберите **New → Blueprint**.
3. Подключите репозиторий.
4. Render прочитает `render.yaml`.
5. Введите секреты:
   - `BOT_TOKEN`
   - `WEBHOOK_SECRET`
6. Нажмите Deploy.

`RENDER_EXTERNAL_URL` Render добавляет автоматически, поэтому отдельно указывать адрес webhook не нужно.

## Важно

Free Web Service на Render засыпает после 15 минут без входящего HTTP-трафика. Telegram webhook разбудит сервис, но первый ответ после сна может задержаться. Для производственного использования рекомендуется платный always-on сервис.

Free Render PostgreSQL предназначен для тестирования и имеет ограниченный срок жизни. Для постоянной истории используйте платный PostgreSQL.
